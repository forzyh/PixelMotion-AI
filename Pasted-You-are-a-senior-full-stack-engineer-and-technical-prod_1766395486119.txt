You are a senior full-stack engineer and technical product designer.
Build a Windows-only desktop app called “PixelMotion AI” that generates pixel sprite motion sheets from a single uploaded pixel character image using AI — and it MUST support both:
(A) Cloud AI (OpenAI) and
(B) Self-hosted / Local AI (offline or LAN server).

GOAL
- User uploads ONE pixel character image (PNG).
- App shows 14 preset motion types and generates a sprite sheet for any selected motion.
- Pixel-only output: crisp pixels, no blur, no anti-aliasing, no painterly style.
- Output: PNG sprite sheet + JSON metadata (frame size, frame count, pivot recommendation).
- Dev runs in Replit. Final build is Windows .exe.

TECH STACK (required)
- Electron (Windows desktop)
- React + TypeScript UI
- Node.js backend (inside Electron main or local Express server started by Electron)
- Post-processing: Node “sharp”:
  - nearest-neighbor scaling
  - optional palette reduction / quantization
- Packaging: electron-builder (Windows NSIS installer)

CRITICAL: AI PROVIDER ABSTRACTION (must implement)
Implement an AIProvider interface so we can switch between providers:

interface AIProvider {
  id: "openai" | "comfyui" | "a1111" | "local-diffusers";
  displayName: string;
  isConfigured(): boolean;
  generateSpriteSheet(inputImagePath, prompt, options): Promise<GeneratedImageResult>;
}

The UI must let the user choose the provider per generation.

AI MODES (must support)
1) OpenAI Mode (cloud)
- Uses OpenAI image generation API.
- Requires OPENAI_API_KEY in Settings.
- Model name is configurable.

2) Self AI Mode (local / offline / LAN)
Support at least ONE of these self AI options (preferably more):
A) ComfyUI HTTP API (recommended)
- User runs ComfyUI locally (127.0.0.1) or on LAN (e.g., 192.168.x.x).
- App sends: reference image + prompt + workflow template.
- Provide Settings fields:
  - ComfyUI server URL
  - Workflow JSON (editable text area) OR built-in workflow templates
  - Optional: ControlNet / IPAdapter toggle fields (if workflow supports)

B) Automatic1111 (Stable Diffusion WebUI) API
- User provides A1111 base URL and API endpoints.
- App sends img2img request: init image + prompt + parameters.
- Settings fields:
  - A1111 URL
  - Sampler, steps, CFG, denoise, width/height
  - Optional: “pixel model” checkpoint name and LoRA names

C) Local Diffusers (embedded Python service) [optional but strongly desired]
- Bundle a minimal Python backend that runs locally (FastAPI) using diffusers.
- The Electron app starts/stops the Python service automatically.
- Settings fields:
  - Model path (local folder)
  - Device: CUDA / DirectML / CPU
  - Optional LoRA folder path
NOTE: This may be heavy. If too large for Replit, implement the integration but document that the model files are user-provided.

PROMPTING RULES (must implement)
- Always generate ONE motion per request.
- Force pixel constraints in every provider:
  - “2D pixel art sprite sheet”
  - “side view, facing right”
  - “crisp pixels, no blur, no anti-aliasing”
  - “consistent character proportions across all frames”
  - “plain transparent background look”
- Force a strict layout:
  - “Arrange frames in a single horizontal row of N frames”
- Motion-specific cues (per preset):

  - Jump Up/Fall: clear arcs and anticipation

  - Hit: recoil -> recover
  - Death: collapse -> final pose

  - Crouch: lower silhouette
  - Climb/Ledge: hang -> pull up
- Allow user “extra instructions” appended at the end.
